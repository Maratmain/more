# Формула важности вопросов для интервью

## Общая формула
```
weight = (criticality × 0.4) + (replaceability × 0.3) + (market_scarcity × 0.2) + (synergy × 0.1)
```

Где все значения в диапазоне 0.0 - 1.0

## Компоненты формулы

### 1. Criticality (Критичность) — 40% веса
**Определение**: Насколько критично знание этого навыка для выполнения основных обязанностей роли.

**Шкала оценки**:
- **0.9-1.0**: Критически важно (без этого навыка невозможно работать)
- **0.7-0.8**: Очень важно (значительно влияет на эффективность)
- **0.5-0.6**: Важно (влияет на качество работы)
- **0.3-0.4**: Желательно (улучшает результаты)
- **0.1-0.2**: Полезно (дополнительный навык)

**Примеры для BA (антифрод)**:
- AntiFraud_Rules: 0.95 (критично для основной функции)
- Requirements_Engineering: 0.9 (ключевая компетенция BA)
- Testing_UAT: 0.75 (важно для качества)
- Documentation: 0.65 (желательно для процессов)

**Примеры для ИТ (ЦОД)**:
- DC_HW_x86_RAID_BMC: 0.95 (критично для работы с серверами)
- LAN_SAN_Networking: 0.9 (основа инфраструктуры)
- Incident_Handling: 0.9 (критично для эксплуатации)
- Tooling_Excel_Visio: 0.6 (полезно для отчетности)

### 2. Replaceability (Заменяемость) — 30% веса
**Определение**: Насколько легко заменить этот навык другими компетенциями или обучить ему.

**Шкала оценки**:
- **0.9-1.0**: Очень сложно заменить (уникальные знания)
- **0.7-0.8**: Сложно заменить (специализированные навыки)
- **0.5-0.6**: Умеренно заменить (требует времени на обучение)
- **0.3-0.4**: Легко заменить (стандартные навыки)
- **0.1-0.2**: Очень легко заменить (базовые навыки)

**Примеры**:
- Специфические антифрод-правила: 0.9 (уникальные знания)
- Настройка RAID: 0.8 (специализированный навык)
- Работа с Excel: 0.3 (стандартный навык)
- Коммуникация: 0.4 (базовый навык)

### 3. Market Scarcity (Дефицит на рынке) — 20% веса
**Определение**: Насколько редко встречается этот навык среди кандидатов на рынке труда.

**Шкала оценки**:
- **0.9-1.0**: Очень редкий навык (менее 10% кандидатов)
- **0.7-0.8**: Редкий навык (10-25% кандидатов)
- **0.5-0.6**: Умеренно редкий (25-50% кандидатов)
- **0.3-0.4**: Распространенный (50-75% кандидатов)
- **0.1-0.2**: Очень распространенный (более 75% кандидатов)

**Примеры**:
- Опыт с BMC/IPMI: 0.9 (очень редкий)
- Знание антифрод-регуляторики: 0.8 (редкий)
- Настройка Windows Server: 0.5 (умеренно редкий)
- Работа с документацией: 0.2 (очень распространенный)

### 4. Synergy (Синергия) — 10% веса
**Определение**: Насколько этот навык усиливает другие компетенции кандидата.

**Шкала оценки**:
- **0.9-1.0**: Высокая синергия (значительно усиливает другие навыки)
- **0.7-0.8**: Умеренная синергия (улучшает общую эффективность)
- **0.5-0.6**: Низкая синергия (небольшое улучшение)
- **0.3-0.4**: Минимальная синергия (незначительное влияние)
- **0.1-0.2**: Отсутствие синергии (изолированный навык)

**Примеры**:
- Знание SQL + Антифрод: 0.9 (высокая синергия)
- Опыт с сетями + Серверы: 0.8 (умеренная синергия)
- Excel + Отчетность: 0.6 (низкая синергия)
- Visio + Документация: 0.4 (минимальная синергия)

## Практические примеры расчета

### Пример 1: AntiFraud_Rules (BA)
```
criticality = 0.95  # Критично для основной функции
replaceability = 0.8  # Сложно заменить специфическими знаниями
market_scarcity = 0.7  # Редкий навык на рынке
synergy = 0.9  # Высокая синергия с другими навыками

weight = (0.95 × 0.4) + (0.8 × 0.3) + (0.7 × 0.2) + (0.9 × 0.1)
weight = 0.38 + 0.24 + 0.14 + 0.09 = 0.85
```

### Пример 2: DC_HW_x86_RAID_BMC (IT)
```
criticality = 0.95  # Критично для работы с серверами
replaceability = 0.8  # Специализированный навык
market_scarcity = 0.8  # Редкий навык
synergy = 0.8  # Умеренная синергия с другими навыками

weight = (0.95 × 0.4) + (0.8 × 0.3) + (0.8 × 0.2) + (0.8 × 0.1)
weight = 0.38 + 0.24 + 0.16 + 0.08 = 0.86
```

### Пример 3: Documentation (BA)
```
criticality = 0.6  # Важно, но не критично
replaceability = 0.3  # Легко заменить
market_scarcity = 0.2  # Очень распространенный
synergy = 0.4  # Минимальная синергия

weight = (0.6 × 0.4) + (0.3 × 0.3) + (0.2 × 0.2) + (0.4 × 0.1)
weight = 0.24 + 0.09 + 0.04 + 0.04 = 0.41
```

## Рекомендации по использованию

### Для генерации вопросов
1. **Анализируй JD**: Выдели ключевые компетенции
2. **Оценивай компоненты**: Присвой значения каждому компоненту
3. **Рассчитывай вес**: Используй формулу для получения итогового веса
4. **Корректируй**: Учитывай специфику роли и компании

### Для валидации весов
1. **Проверяй логику**: Веса должны соответствовать важности навыков
2. **Сравнивай с эталонами**: Используй примеры из успешных интервью
3. **Тестируй на практике**: Корректируй веса на основе результатов

### Для оптимизации
1. **Мониторь результаты**: Отслеживай эффективность вопросов
2. **Обновляй веса**: Адаптируй под изменения в требованиях
3. **Учитывай контекст**: Корректируй под специфику компании

## Важные замечания

- **Контекст важен**: Веса могут отличаться для разных компаний
- **Время изменяется**: Редкость навыков может меняться со временем
- **Баланс**: Не все вопросы должны иметь высокий вес
- **Гибкость**: Формула — это ориентир, а не жесткое правило
